pytest框架
发表于 2020-09-13| 更新于 2020-09-13
阅读量:

安装与入门

安装pytest

  • 安装：
    • pip install -u pytest
  • 查看是否安装及其版本号：
    • pytest –version

使用和调用

可能退出的代码

  • Exit code 0：所有测试都已收集并成功通过
  • Exit code 1：测试已收集并运行，但有些测试失败
  • Exit code 2：测试执行被用户中断
  • Exit code 3：执行测试时发生内部错误
  • Exit code 4：pytest命令行使用错误
  • Exit code 5：未收集任何测试

获取帮助信息

  • 查看pytest版本：pytest –version
  • 显示可用的内置函数参数：pytest –fixtures
  • 在命令行和配置文件选项中显示帮助：pytest -h | –help

第一次或n次故障后停止

  • pytest -x：出现1个失败就停止测试
  • pytest –maxfail=2：出现2个失败就停止测试

指定测试/选择测试

  • pytest支持几种方法来运行和从命令行中选择测试

在模块中运行测试

  • pytest test_mod.py

在目录中运行测试

  • pytest testing/

按关键字表达式运行测试

  • pytest -k “MyClass and not method”
  • 这命令会去匹配文件名、类名和函数名的用例。上面的示例将运行TestMyClass.test_something 但不是TestMyClass.test_method_simple。

按节点ID运行测试

  • 每个收集的测试都被分配一个唯一的节点id(node id)，它由模块文件名和诸如类名、函数名和参数化等说明组成，用::字符。

在模块内运行特定测试：

  • pytest test_mod.py::test_func

在命令行中指定测试方法的另一个示例：

  • pytest test_mod.py::TestClass::test_method

通过标记表达式运行测试

  • pytest -m slow：将运行所有用@pytest.mark.slow装饰符

从包运行测试

  • pytest –pyargs pkg.testing：将导入pkg.testing并使用文件系统位置来查找和运行测试。

多进程运行cases

  • 在测试用例量很多时， 运行事件也会变的很长，如果想缩短脚本运行的时长，就可以用多进程来运行/
  • 安装pytest—xdist：pip install -U pytest-xdist
  • 运行模式：pytest test_se.py -n NUM （NUM等于并发的进程数）

重试运行cases

  • 当用例因为超时或其他原因导致运行失败的时候，我们可以通过重试运行cases的方式来解决。
  • 安装pytest-rerunfailures：pip install -U pytest-rerunfailures
  • 运行模式：pytest test_se.py –reruns NUM （NUM等于重试的次数）

显示print内容

  • 在运行pytest的时候，print内容不会被显示出来。但如果加上-s，则可以显示出来。
  • 运行模式： pytest test_se.py -s
  • 在pytest的多进程运行模式时可以叠加执行。如：同时运行4个进程+打印print内容：
  • pytest test_se.py -s -n 4

Skip and xfail：处理无法成功的测试

测试中断言的编写和报告

断言assert

  • pytest允许使用标准的python中的assert来验证测试中的预期结果和实际结果。
  • assert用于判断一个表达式，在表达式条件为false的时候触发异常。
  • 断言可以在条件不满足程序运行的情况下直接返回错误，而不必等待程序运行后出现崩溃的情况。
  • 语法格式：
    • assert expression等价于：if not expression:raise AssertionError
    • assert expression [，arguments]等价于：if not expression:raise AssertionError(arguments)
